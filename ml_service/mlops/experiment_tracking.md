# Experiment Tracking Strategy

To ensure 100% reproducibility, every training run in Seed2Success is recorded in an Experiment Tracker (conceptually modeled after MLflow/Weights & Biases).

## 1. What is Tracked?
For every execution of `train.py`, we log:

### A. Environment & Code
- **Git Commit Hash:** To exactly replicate the code state.
- **Environment:** Python version, dependency hashes (requirements.txt/Poetry).
- **Engineer/Owner:** Who triggered the run.
- **Random Seed:** Crucial for reproducing Random Forest splits.

### B. Parameters
- **Hyperparameters:** `n_estimators`, `max_depth`, `min_samples_split`.
- **Dataset Version Hash:** E.g., `dataset_v1_8a9b2f`.

### C. Metrics
- **Training Time (seconds)**
- **Accuracy, F1-Score, R², MAE, RMSE**
- **Validation Score vs Training Score** (To detect overfitting).

### D. Artifacts
- The raw pickled model (`best_model.pkl`).
- SHAP dependency plots (PNGs).
- Feature Importance arrays.

## 2. The Golden Rule
"If an experiment is not tracked, it never happened." Data Scientists are not allowed to push local `.pkl` files to production. All models must be generated by the tracked CI pipeline.
